# MEMORY

## 项目管理规范
- **分支策略**: 每个项目独立分支 (`作者/项目名`)。新项目创建时使用 `--orphan` 模式以确保不继承 `main` 分支的无关文件（如 `daily/` 目录）。现有分支严禁清理历史。
- **禁止污染**: 项目分支严禁出现 `daily/`、`.workbuddy/` 等全局管理文件。
- **Git 规范**: 保持历史清爽，使用中文 Commit Message。`origin` 为用户 fork，`upstream` 为源仓库。

## 调研笔记结构
- **位置**: 分支根目录 `README.md`。
- **结构**: 1. 项目背景; 2. 核心技术/架构; 3. 亮点与创新; 4. 调研结论。

## 调研日报规范
- **存储**: `daily/YYYY-MM-DD.md`。
- **结构**: 概览、详情（含 Stars/笔记链接）、仓库治理记录。

## 环境偏好
- **源码位置**: `~/playground/` 目录，添加 `🦞` 标签。
- **自动化工具**: `scripts/record_note.py` (自动化录入与索引同步)。
